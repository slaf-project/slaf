    The Single-Cell Format for the Virtual Cell Era¶

    Github: github.com/slaf-project/slaf Docs: slaf-project.github.io/slaf/

    The Promise of the Virtual Cell¶

    Figure 1. The Virtual Cell: a unified, queryable representation of all single-cell data, enabling prediction, explanation, and discovery.
    Figure 5. Virtual Cells power new applications: prediction, explanation, and discovery across tissues, modalities, and experiments.
    • Virtual Cells unify all single-cell data into a single, queryable format.
    • Enable prediction of cell states, explanation of biological phenomena, and discovery of new cell types.
    • Break down barriers between datasets, tissues, and modalities.

    Current Formats Are Not Cloud-Native¶

    Traditional single-cell workflows hit fundamental bottlenecks:¶

    • Network: Days to transfer 100M-cell datasets
    • Memory: Out-of-memory errors on standard hardware
    • Infrastructure: Requires dedicated engineering teams
    • Duplication: Multiple copies per experiment/node

    SLAF: Bringing Cloud-Native to Single-Cell¶

    Built on Proven Technologies¶

    • Lance: Cloud-native columnar storage
    • Polars: High-performance dataframe engine
    • DuckDB: Embedded OLAP database
    • Dask: Lazy computation graphs

    SLAF combines these innovations for single-cell data¶

    SLAF Architecture¶

    Three Core Innovations¶

    1. OLAP-Powered SQL¶

    Embedded, in-process query engines for pushdown optimization

    2. Memory Efficient¶

    Lazy evaluation - only load what you need

    3. Scanpy Compatible¶

    Drop-in replacement with familiar numpy-like slicing

    SLAF: Not Just Storage, But Experience¶

    "You Never Left Scanpy"¶

    from slaf.integrations import read_slaf

# Load as lazy AnnData
adata = read_slaf("s3://bucket/large_dataset.slaf")

# Operations are lazy until you call .compute()
subset = adata[adata.obs.cell_type == "T cells", :]
first_ten_cells = subset[:10, :]
expression = first_ten_cells.X.compute()  # Only now is data loaded

    Leverage Pushdown Filtering¶

    **Do this:** 
    from slaf import SLAFArray

slaf_array = SLAFArray("s3://bucket/large_dataset.slaf")

results = slaf_array.query("""
SELECT cell_type, AVG(total_counts) as avg_counts
FROM cells
WHERE batch = "batch1"
AND cell_type IN ("T cells", "B cells")
GROUP BY cell_type
ORDER BY avg_counts DESC
""")
