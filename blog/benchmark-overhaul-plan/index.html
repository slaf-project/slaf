    SLAF Benchmark Overhaul Plan¶

    Overview¶

    This document outlines our plan to completely overhaul the SLAF benchmarks to demonstrate our competitive advantages against state-of-the-art dataloaders like scDataset, BioNeMo SCDL, and AnnDataLoader.

    Key Competitive Advantages¶

    SLAF demonstrates significant advantages over state-of-the-art dataloaders:

    1. 4.2x Higher Throughput: 16,789 vs 4,000 cells/sec
    2. 100x Lower Memory: 2.5 GB vs 256 GB RAM
    3. Single Process: No multiprocessing overhead
    4. GPU-Ready Output: Pre-tokenized sequences vs raw data
    5. Inferior Hardware: M1 Max laptop vs enterprise server

    Benchmark Structure¶

    Class 1: Internal SLAF Performance Analysis¶

    Goal: Show SLAF's performance across different tokenization strategies using Tahoe100M dataset.

    Scope:

    • Single batch size (32 cells) - most common in literature
    • 4 tokenization strategies:
    • scGPT with binning
    • scGPT without binning
    • Geneformer with percentile filtering
    • Geneformer without percentile filtering
    • No multi-process scaling or batch size scaling
    • Focus on throughput, memory, and tokenization quality

    Key Metrics:

    • Throughput (cells/sec, tokens/sec)
    • Memory efficiency (RAM usage)
    • Tokenization strategy comparison
    • Batch processing efficiency

    Expected Results:

    • Peak throughput: ~16,789 cells/sec (Geneformer xlarge batch)
    • Token generation: Up to 34M tokens/sec for large batches
    • Memory efficiency: Linear scaling with batch size (~1.2MB per 512 cells)
    • Strategy advantage: Geneformer shows ~10% higher throughput than scGPT

    Class 2: External Dataloader Comparisons¶

    Goal: Compare SLAF against state-of-the-art dataloaders using standardized benchmark setup.

    Two-Tier Comparison:

    Tier 1: Raw Data Loading (cells/sec)¶

    Compare raw data loading performance across systems:

    • SLAF: Raw cell × gene data (new raw mode)
    • scDataset: Raw cell × gene data
    • AnnDataLoader: Raw cell × gene data
    • BioNeMo SCDL: Raw cell × gene data

    Metrics:

    • Throughput (cells/sec)
    • Memory usage (GB)
    • Output format consistency
    • Resource requirements

    Tier 2: GPU-Ready Output Comparison¶

    Compare end-to-end pipeline performance:

    • SLAF: Raw data → GPU-ready tensors (tokenized sequences)
    • Others: Raw data only (requires manual tokenization)

    Metrics:

    • End-to-end throughput
    • Computational output quality
    • Training integration readiness
    • Manual processing requirements

    Implementation Plan¶

    Phase 1: Add Raw Mode to SLAFDataLoader¶

    Goal: Implement raw data mode for fair comparisons.

    Changes Required:

    1. Add raw_mode: bool = False parameter to SLAFDataLoader.__init__()
    2. Implement _raw_iterator() method that bypasses tokenization
    3. Return raw cell × gene data in consistent format
    4. Maintain existing API compatibility

    Raw Mode Output Format:

    We will match AnnDataLoader's sparse tensor format for consistency:

    {
    "X": torch.sparse_csr_tensor(
        crow_indices=tensor([0, 4717, 5733, ...]),  # Row pointers

        size=(batch_size, n_genes),                  # Matrix dimensions
        nnz=46260,                                   # Non-zero elements
        layout=torch.sparse_csr

}

dataloader = DataLoader(sc_dataset, batch_size=None)

    AnnDataLoader Integration:

    from scvi.dataloaders import AnnDataLoader
from scvi.data import AnnDataManager
from scvi.data.fields import LayerField

anndata_fields = [LayerField(registry_key="X", layer=None, is_count_data=True)]
adata_manager = AnnDataManager(fields=anndata_fields)
adata_manager.register_fields(adata)

dataloader = AnnDataLoader(
    adata_manager,
    batch_size=32,
    shuffle=True,
    drop_last=False,
    load_sparse_tensor=True

    Expected Outcomes¶

    Competitive Positioning¶

    Raw Data Loading (Tier 1):

    • SLAF: ~8,000 cells/sec (estimated)
    • scDataset: 4,000 cells/sec
    • AnnDataLoader: ~500 cells/sec
    • Result: 2x faster than scDataset, 16x faster than AnnDataLoader

    GPU-Ready Output (Tier 2):

    • SLAF: 16,789 cells/sec (tokenized sequences)
    • Others: Raw data only (requires manual processing)
    • Result: 4.2x faster + GPU-ready output

    Key Messages¶

    1. Performance: SLAF achieves higher throughput with fewer resources
    2. Efficiency: Single process vs multiprocessing, lower memory usage
    3. Functionality: GPU-ready output vs raw data requiring manual processing
    4. Hardware: Inferior hardware achieving superior results
    5. Architecture: Streaming design enables massive dataset handling
